방안 1 정적 데이터 스냅샷 파이프라인 (JSON Export)
운영 WAS가 배치로 JSON 스냅샷을 사내 저장소에 push → 빌드 서버는 저장소에서만 데이터를 읽어 SSG 수행
데이터 플로우
  1. ① 운영 WAS의 Export Job이 스냅샷 JSON을 사내 저장소로 단방향 push (운영→빌드망 방향만 허용)
  2. ② 빌드 서버가 react-router.config.ts의 prerender에서 저장소 URL을 fetch
  3. ③ SSG HTML 생성 후 CDN 배포 → 사용자에게 서빙

방안 3 SSG Shell + 런타임 로딩 + Playwright 크롤러 하이브리드
빌드 시엔 데이터 없는 Shell HTML만 생성 · 배포 후 Playwright 크롤러가 실제 렌더링된 HTML로 덮어씀 · 런타임 clientLoader가 WAS 데이터 로드
데이터 플로우
  1. ① 빌드 서버가 WAS 호출 없이 Shell HTML(레이아웃·스켈레톤)만 prerender하여 CDN 배포
  2. ② 배포 후 Playwright Crawler가 실제 도메인을 렌더링하여 데이터가 채워진 HTML 생성
  3. ③ Rendered HTML을 CDN에 덮어씀 → 검색엔진 봇/초기 접속자는 완성본 수신
  4. ④ 사용자 접속 시 CDN에서 HTML 서빙 (Shell 또는 Rendered)
  5. ⑤ Hydration 후 clientLoader가 운영 WAS를 직접 호출하여 최신 데이터 확보
  6. ⑥ 최신 데이터로 화면 갱신 → 실시간성 확보
